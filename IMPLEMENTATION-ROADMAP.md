# Implementation Roadmap

## Step 1 — Confirm commercial identity

Choose brand/domain, legal company name, sales email, WhatsApp, target countries, currency policy, MOQ, warranty, delivery terms, and languages. Do not reuse FJINNO contact or brand details unless this site is legally the same business.

## Step 2 — Freeze the product matrix

Create one approved spreadsheet for every model with range, accuracy, outputs, switch quantity/rating, supply, bulb, thread, capillary, protection, dimensions, environment, standards, options, and compatible remote display.

## Step 3 — Curate media

Review the downloaded source images, remove duplicates/thumbnails, confirm usage rights, rename semantically, create WebP/AVIF variants, and add exact alt text. Request clean front, side, terminal, sensor bulb, capillary, packaging, installation, and factory-test photos.

## Step 4 — Build the MVP

Build an Astro static site with the pages in `SITE-STRUCTURE.md`. MVP target: 6 product pages, 3 category pages, 3 application pages, 4 technical guides, and essential company/legal pages.

## Step 5 — Add RFQ conversion

Use compact inquiry blocks and a full RFQ page. Add spam protection, success/error states, file upload or drawing-request workflow, email delivery, and analytics events. Never expose API keys in frontend code.

## Step 6 — Add technical SEO

Implement titles, descriptions, canonical URLs, breadcrumbs, Product schema, Organization schema, XML sitemap, robots.txt, Open Graph, image sitemap entries where appropriate, and redirects. Validate all URLs in a production build.

## Step 7 — QA

- Verify every specification against approved source data
- Test desktop/mobile navigation and all RFQ paths
- Test email delivery and spam controls
- Check accessibility, keyboard use, contrast, and form labels
- Run Lighthouse and fix LCP/CLS issues
- Confirm no copied FJINNO branding/contact details remain

## Step 8 — Launch and measure

Connect the domain, deploy to Cloudflare Pages, submit sitemap to Google and Bing, and monitor indexing, RFQ events, query impressions, and landing-page conversion.

## Step 9 — Grow topical authority

Publish installation, comparison, wiring, troubleshooting, and verified case content. Update model pages whenever drawings or technical data change.

## Recommended immediate next action

Before visual design, complete the approved product matrix and provide the new site's brand/domain/contact details. Those choices determine page copy, RFQ routing, schema, and legal footer content.
